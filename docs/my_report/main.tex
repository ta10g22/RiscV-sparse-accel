\begin{abstract}
Sparse matrix--dense matrix multiplication (SpMM) is a key kernel in
graph neural networks, recommender systems, and compressed neural
network inference, but it performs poorly on general-purpose processors
because of irregular memory access and low arithmetic intensity. This
report presents the design, implementation, and evaluation of a PicoRV32
RISC-V system-on-chip with a custom hardware accelerator for compressed
sparse row (CSR) SpMM on the Terasic DE1-SoC Cyclone~V FPGA. The design
was developed in three optimisation stages: a baseline accelerator with
tile length ($TN = 8$) parallel MAC lanes across the dense output
dimension, a DSP-oriented MAC enhancement that maps each lane to a
dedicated Cyclone~V multiplier-accumulator block, and a runtime
symmetric INT8 packing stage that stores four quantised operands per
32-bit RAM word. A 12-case hardware benchmark suite evaluating matrix
size, dense width, sparsity level, and nonzero pattern was executed in
firmware on the synthesised SoC, and all cases passed element-by-element
correctness checks. The final INT8 design achieved a peak speedup of
$56.80\times$ over the quantised PicoRV32 softcore and $11.92\times$
over an ARM Cortex-M0 softcore with a $2.87\times$ cycle reduction over
baseline.
\end{abstract}

\chapter[Appendix A: SpMM Background]{SpMM Background}
\label{app:spmm-background}

This appendix provides supporting background on the compressed sparse row (CSR)
format and its role in SpMM, for readers less familiar with sparse linear algebra.

\section*{CSR Format Details}

A sparse matrix $A$ of size $M \times K$ with NNZ nonzeros is stored in CSR
as three arrays:
\begin{itemize}
  \item \texttt{rowptr[0..M]}: an array of $M + 1$ integers, where
        \texttt{rowptr[i]} is the index into \texttt{colidx} and \texttt{values}
        of the first nonzero in row $i$. By convention, \texttt{rowptr[M] = NNZ}.
  \item \texttt{colidx[0..NNZ-1]}: the column index of each nonzero.
  \item \texttt{values[0..NNZ-1]}: the value of each nonzero.
\end{itemize}

The nonzeros of row $i$ are located at indices \texttt{rowptr[i]}
through \texttt{rowptr[i+1] - 1} in both \texttt{colidx} and \texttt{values}.
The number of nonzeros in row $i$ is \texttt{rowptr[i+1] - rowptr[i]}.

\section*{Symmetric Quantisation Detail}

For a vector $\mathbf{v}$ with maximum absolute value $v_{\max}$, symmetric
INT8 quantisation maps each element $v_i$ to a signed 8-bit integer:
\[
  q_i = \mathrm{clip}\!\left(\mathrm{round}\!\left(\frac{v_i \cdot 127}{v_{\max}}\right),
        -127, 127\right).
\]
The scale factor $s = v_{\max} / 127$ allows reconstruction:
$\hat{v}_i = q_i \cdot s$. The maximum absolute quantisation error is
$s / 2 = v_{\max} / 254$.

Symmetric quantisation does not require a zero-point offset (the zero value
maps to zero exactly), which simplifies the accumulation logic in hardware.

\chapter[Appendix C: Benchmark Notes]{Benchmark Notes and Stage Relationship}
\label{app:bench-notes}

This appendix records the relationship between the evaluation stages discussed
in the main report:
\begin{itemize}
  \item the \textbf{baseline accelerator} and \textbf{parallel DSP MAC}
        measurements are historical stage results collected using firmware builds
        before INT8 packing became the default mainline path;
  \item the \textbf{INT8} stage is the current mainline hardware/software
        path in the repository; and
  \item the \textbf{Cortex-M0} comparison reuses the same benchmark definitions
        and the same INT8 accelerator reference cycles, making it a second software
        baseline, not a different hardware experiment.
\end{itemize}

The benchmark IDs T1--T12, their matrix dimensions, sparsity levels, and pattern
definitions are fixed across all stages. All CPU and accelerator cycle counts
reported in Chapter~\ref{ch:results} were collected using the PicoRV32
\texttt{rdcycle} counter on a physical DE1-SoC board programmed with the
synthesised SoC bitstream.

\chapter[Appendix D: Design and Data Archive]{Design and Data Archive}
\label{app:archive}

The complete project repository is organised as follows:
\begin{itemize}
  \item \texttt{rtl/}: accelerator RTL, comprising the top-level wrapper
        (\texttt{accel\_top.sv}), the control finite state machine
        (\texttt{accel\_ctrl.sv}), and the parallel MAC datapath
        (\texttt{accel\_datapath.sv}).
  \item \texttt{fpga/rtl/}: synthesisable SoC top-level
        (\texttt{soc\_top.sv}) instantiating PicoRV32, the dual-port BRAM,
        the accelerator, the UART peripheral, and the GPIO interface.
  \item \texttt{fpga/quartus/}: Quartus project files
        (\texttt{de1\_soc\_spmm.qpf}), pin assignment constraints
        (\texttt{de1\_soc\_spmm.qsf}), timing constraints
        (\texttt{de1\_soc\_spmm.sdc}), and the byte-lane memory
        initialisation files (\texttt{firmware[0-3].mif}) generated from
        the compiled firmware image.
  \item \texttt{sw/driver/}: bare-metal firmware, including the RISC-V
        assembly startup (\texttt{start.S}), benchmark harness
        (\texttt{main.c}), accelerator driver
        (\texttt{spmm\_accel.c/.h}), libc stubs
        (\texttt{libc\_stubs.c}), linker script (\texttt{linker.ld}),
        \texttt{Makefile}, and the Python helper scripts
        (\texttt{bin2hex32.py}, \texttt{hex2mif.py}) that convert the
        compiled ELF into the byte-lane MIF files consumed by Quartus.
  \item \texttt{tb/common/}: shared testbench infrastructure, comprising
        the clock/reset generator (\texttt{clk\_reset.sv}), the MMIO
        SystemVerilog interface (\texttt{mmio\_if.sv}), the shared
        utility package (\texttt{tb\_pkg.sv}), the assertion macros
        (\texttt{tb\_macros.svh}), and the shared MMIO register-map
        header (\texttt{tb\_regmap.svh}).
  \item \texttt{tb/unit/}: unit testbenches for the individual
        accelerator modules (\texttt{accel\_ctrl\_tb.sv},
        \texttt{accel\_datapath\_tb.sv}, \texttt{accel\_top\_tb.sv}).
  \item \texttt{tb/integ/}: integration testbench (\texttt{soc\_tb.sv})
        that exercises the full synthesisable SoC.
  \item \texttt{sim/}: simulator command scripts
        (\texttt{compile\_soc.do}) for invoking the testbenches under
        ModelSim/Questa.
  \item \texttt{ip/}: third-party IP cores used unmodified, namely the
        PicoRV32 RV32IM soft core (\texttt{ip/picorv32/}) and the
        Cortex-M0 reference source (\texttt{ip/arm\_m0/}) used to derive
        the Cortex-M0 cycle estimates reported in
        Table~\ref{tab:cortex-results}.
  \item \texttt{docs/my\_report/}: report \LaTeX\ source, bibliography
        (\texttt{bib/ECS.bib}), figure assets, and chapter files under
        \texttt{chapters/}.
  \item \texttt{README.md}: top-level repository overview, build
        instructions, and quick-start guide for reproducing the
        benchmark results.
\end{itemize}

All source, testbench, firmware, and Quartus project files required to
reproduce the measurements reported in Chapter~\ref{ch:results} are
contained within this repository.
